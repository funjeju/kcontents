# 07. 시나리오 작성 시스템

큐레이터(운영팀)가 시나리오를 추가할 수 있는 워크플로우, 도구, 체크리스트.

**핵심 원칙:** 작가가 시나리오를 *기획*하면, 명장면 narrative는 AI가 *생성*한다. 사람의 일은 setup과 outcomes 정의까지.

---

## 1. 시나리오 작성의 핵심 철학

### 1.1 드라마 IP를 가져오는 것이 아니라, 시대를 가져온다

K-Drama Life의 시나리오는 *특정 드라마를 게임화*하는 것이 아니다. 그 드라마가 *그리는 시대*를 게임화하는 것이다.

```
❌ 잘못된 접근
"미스터 션샤인을 게임으로 만들자"
- 캐릭터: 유진 초이, 고애신, 김희성, 구동매
- 사건: 드라마 회차별 사건들
- 결과: IP 침해 위험, 드라마 본 사람만 즐길 수 있음

✅ 올바른 접근
"1900년대 한성의 사회를 살아보게 하자"
- 캐스팅: 미국 귀환 군관, 양반가 도련님, 의병 처녀, 야쿠자, 거리의 목격자
- 사건: 실제 역사적 사건 (정미의병, 헤이그 밀사, 한일강제병합)
- 결과: 합법, 누구나 즐김, K-드라마 팬은 정서 알아챔
```

### 1.2 작가의 역할 = 시대 큐레이터

큐레이터가 시나리오를 추가할 때 하는 일:

1. **시대 선택** — 어느 연도 어느 사회를 다룰지
2. **무대 설정** — 그 시대의 일상, 갈등, 변화
3. **캐스팅 풀 설계** — 5명의 가능한 인물 정의
4. **명장면 setup 작성** — *상황*만 정의, narrative는 AI가
5. **결말 분기 설계** — 각 캐스팅마다 6~10개 결말
6. **위치 카드 매핑** — 실제 장소 연결
7. **메타 챕터 사실 검증** — 무거운 사건의 출처

## 2. 시나리오 작성 워크플로우

### 2.1 Phase 1: 시나리오 컨셉 (30분)

**작성하는 것:** Scenario Concept Document (1페이지)

```markdown
# [시나리오 가칭]

## 시대
- 연도: 1894 ~ 1907
- 지역: 한성 (조선/대한제국)
- 사회 변화: 갑오개혁 → 대한제국 → 일제강점

## 정서적 모티프
- 격동의 시대를 평범한 사람이 어떻게 살아남았는가
- 양반과 평민의 신분 해체 과정
- 외세의 그늘 아래 정체성

## 비교 K-드라마 정서 (참고용, 인용 X)
- 미스터 션샤인 — 1900년대 한성, 의병
- 도시-남자친구 — (없음)
- 동주 영화 — (다음 시기)

## 캐스팅 풀 (5명, 임시)
1. 이방인의 길 (미국 귀환 군관 격)
2. 양심의 도련님 (친일가 도련님 격)
3. 의기 있는 처녀 (양반가 의병 처녀 격)
4. 백정의 분노 (하층 출신 야쿠자 격)
5. 거리의 목격자 (평민 일상 격) ← 기본값

## 핵심 역사 사건 (챕터 배경)
- 1894 갑오개혁
- 1895 을미사변
- 1897 대한제국 선포
- 1905 을사조약
- 1907 정미의병

## 무게 (heaviness)
- 4/5 (식민지 직전, 폭력 묘사 있음)

## 추천 연령
- 18세 이상

## 예상 결말 풀
- 캐스팅 5종 × 평균 7개 = 35개
```

### 2.2 Phase 2: 챕터 구조 (1시간)

**작성하는 것:** Chapter Skeleton (10개 챕터)

각 챕터는 다음 정보:

```markdown
## Chapter {n}: {age}세, {year}년

### 배경
- 시대 사건: 갑오개혁 마무리, 단발령 강제
- 캐릭터 일상: 한약방 일을 익힘
- 가능한 NPC: 부친, 형, 양반가 따님 서희, 일본 상인

### 이벤트 풀 (8~12개 후보, 챕터당 5~7개 노출)

이벤트 1: "단발령" 
- 종류: branch
- 게이트: philosophy
- 트리거: 누구에게나 노출
- 작가 지시문: "...단발령으로 인한 가족의 충돌 장면..."
- 옵션 A 결과: 도덕성 +2, 부친과의 인연 -1
- 옵션 B 결과: 도덕성 -1, 친일 변수 +1
- 옵션 C: 자유 입력 (AI 평가)

이벤트 2: "서희와의 첫 대화"
- 종류: branch
- 게이트: knowledge
- 트리거: 누구에게나 노출
- ...

이벤트 3: "한약 손질 미션"
- 종류: narrative
- 게이트: creation
- 트리거: 사회성 ≥ 8
- ...

이벤트 4: "일본 상인의 호객"
- 종류: branch
- 게이트: philosophy
- 트리거: 청계천 거주
- ...

(이런 식으로 8~12개)
```

### 2.3 Phase 3: 캐스팅 조건 정의 (30분)

**작성하는 것:** Casting Pool Conditions

```markdown
## 캐스팅 풀 조건

### A. 이방인의 길 (the_returner)
T-0 시점 결정 조건:
- stats.intellect ≥ 14
- qualities.foreign_path ≥ 3
- qualities.father_progressive ≥ 2

설명: 미국 선교사 + 영어 + 진보적 부친 조합이 만든 길.

### B. 양심의 도련님 (the_gentleman)
- 가족 배경: yangban
- stats.morality ≥ 14
- qualities.father_pro_japan ≥ 3
- qualities.uigi ≥ 2

설명: 양반 + 도덕성 + 친일가 + 양심의 갈등 조합.

### C. 의기 있는 처녀 (the_lady)
- 성별: 여성
- 가족 배경: yangban
- stats.intellect ≥ 12
- qualities.uigi ≥ 5

설명: 양반 처녀 + 학식 + 의기.

### D. 백정의 분노 (the_blade)
- 가족 배경: lower_class
- stats.physique ≥ 13
- qualities.rage ≥ 5

설명: 하층 출신 + 강함 + 분노.

### E. 거리의 목격자 (the_witness) ← 기본값
- 위 4개 조건 모두 미충족 시

설명: 평민의 일상.
```

### 2.4 Phase 4: 명장면 setup (1~2시간)

**가장 중요한 작업.** 시나리오당 5~10개 명장면.

**핵심: narrative 텍스트는 작성하지 않는다. AI가 생성한다.**

작성 템플릿:

```markdown
## Iconic Moment: 첫 만남

### 메타
- ID: ms-iconic-01
- 적용 캐스팅: the_returner, the_gentleman
- 챕터 나이: 17세

### 트리거 조건
- relationships.seohi ≥ 3 (사용자가 12~14세에 서희와 인연 만들었음)
- pathVariables.isOnDramaPath = true (드라마 길에 가까운 경우)

### Setup
- 위치: 1902년 봄, 한성 어느 카페 (글로리아 호텔 카페 격)
- 등장 NPC: yangban_lady (서희와 비슷한 결의 인물)
- 정서 톤: 차가운 첫 만남, 호감도 분기, 시대의 무게
- 작가 지시문 (AI에게):
  
  "사용자 캐릭터가 카페에서 책을 읽고 있다.
   양반가 처녀가 들어와 그를 본다.
   그녀의 시선은 차갑다 — 이 시대 양반가 처녀가 외국 군관에게 보내는 시선이다.
   그녀는 묻는다: '미국 해병대 군관이라 들었습니다. 조선 사람이라고도 들었습니다. 어느 쪽입니까?'
   
   AI는 위 상황을 150~200자의 시대극 narrative로 풀어내고,
   다음 3개 옵션을 자연스러운 문장으로 작성:
   
   - 옵션 A (드라마 일치): '둘 다입니다, 미국인이자 조선인.'
   - 옵션 B (갈라진 길): '이방인입니다, 양쪽 모두에.'
   - 옵션 C: 자유 입력"

### Outcomes (작가 정의)

#### 옵션 A 선택 시 (드라마 일치)
- 호감도: yangban_lady +3
- 변수: drama_path_consistent +1
- 스탯: emotion +1, sociability +1

#### 옵션 B 선택 시 (갈라진 길)
- 호감도: yangban_lady +1
- 변수: divergent_path 활성화
- 스탯: intellect +2, sociability -1

#### 옵션 C 선택 시 (자유 입력)
- AI 평가 프롬프트:
  "사용자 답변이 시대극 톤에 적합한지 평가.
   답변에서 보이는 가치관(이방인성, 정체성)을 분석.
   적합한 스탯/변수 변화 추천 (±1~3 범위)."
```

### 2.5 Phase 5: 결말 풀 정의 (1~2시간)

캐스팅 5종 × 평균 7개 = 35개 결말.

각 결말은 다음 정보:

```markdown
## Ending: 도화서 보조 화원

### 메타
- ID: hidden-court-end-01
- 캐스팅: the_painter
- 시나리오: hidden_court

### 결정 조건
- stats.creativity ≥ 16
- stats.morality ≥ 12
- qualities.royal_recognition ≥ 5

### 결말 컨텍스트 (AI에 전달)
- 나이: 22세 (Main Story 종료 시점, 시나리오마다 다름)
- 시대: 1707년 즈음
- 역사 사건: 숙종 시대, 인현왕후 복위 후
- 주요 모티프:
  - 무명 여화원의 길
  - 신분과 성별의 한계 안에서의 자기성취
  - 시대가 인정 못하는 천재의 고독
- 명대사 후보 주제:
  - "내 붓에는 두려움이 없다"
  - "역사는 내 이름을 잊을 것이다"
  - "이 그림이 내가 살았다는 증거다"

### 카드 일러스트 힌트
- 팔레트: joseon_classical
- 구도: 인물 측면, 한지에 붓을 든
- 분위기: 고요한 결의, 고독, 빛

### 희귀도 예상
- 0.5% ~ 2% (까다로운 조건)
```

### 2.6 Phase 6: 위치 카드 매핑 (30분)

명장면 + 챕터별 등장 장소를 실제 위치와 매핑.

```markdown
## Location Card: 정동의 봄

- 시나리오: mr_sunshine
- 트리거 명장면: ms-iconic-01

- 실제 장소: 정동길, 서울 중구
- 좌표: [37.5663, 126.9744]
- 인증 반경: 200m

- 보상:
  - 다음 분기점에서 호감도 +5
  - "한국과의 인연" 변수 +3

- 외부 링크:
  - Google Maps: https://maps.google.com/...
  - Street View: https://...
  - 실시간 풍경 스트림: (해당 위치 라이브 영상이 있다면)
```

### 2.7 Phase 7: 메타 챕터 사실 검증 (30분~1시간)

무거운 챕터에 대한 사실 데이터셋 작성.

```markdown
## Meta Chapter: 관동대지진 학살

### 트리거
- 챕터: chapter_1923_kanto

### 사실 데이터 (RAG로 LLM에 전달)

- 사건명: 관동대지진과 조선인 학살
- 발생 일시: 1923년 9월 1일~9월 중순
- 사건 요약:
  - 1923년 9월 1일 일본 간토 지방 대규모 지진 발생
  - 지진 직후 일본인 사회에서 "조선인이 우물에 독을 풀었다", "조선인이 폭동을 일으킨다"는 유언비어 확산
  - 일본 군경, 자경단에 의해 약 6,000명 추정의 조선인 학살 (정확한 수 미확정)
  - 일본 정부의 묵인 또는 적극 가담 정황
  - 후일 일본에서 오랫동안 부정·축소되었으나, 점차 학계에서 인정

- 출처:
  - Wikipedia (한국어): https://ko.wikipedia.org/wiki/...
  - 한국학중앙연구원: ...
  - Yamada Shoji "관동대지진과 조선인 학살" 책: ...
  - 일본 internal sources: ...

- 회고 질문:
  "당신의 캐릭터는 이 사건을 어떻게 기억할까요?"
```

## 3. 시나리오 작성 도구

### 3.1 작가용 어드민 페이지 (Phase 2 개발)

```
/admin/scenarios/

대시보드:
- 작성 중 시나리오 리스트
- 각 시나리오 진행률 (Phase 1~7 중 어디까지)
- 미리보기 / 테스트 플레이 버튼

시나리오 편집기:
- Phase별 입력 폼
- 캐스팅 풀 visual builder
- 명장면 setup 작성 폼 (AI 미리보기 가능)
- 결말 풀 매트릭스 뷰
- 위치 카드 지도 마커 도구

테스트 플레이:
- 작가가 자기 시나리오를 테스트
- 스탯/변수 cheat (조건 만족 강제)
- 모든 명장면 / 결말 도달 시뮬레이션
```

### 3.2 시나리오 템플릿 파일

작가는 마크다운 + JSON 파일을 직접 편집할 수도 있다 (Phase 1):

```
/scenarios-content/{scenarioId}/
├── 00-concept.md           # Phase 1 컨셉
├── 01-chapters.md          # Phase 2 챕터 구조
├── 02-casting.json         # Phase 3 캐스팅 풀
├── 03-iconic-moments/      # Phase 4 명장면들
│   ├── ms-iconic-01.json
│   └── ...
├── 04-endings/             # Phase 5 결말들
│   ├── ending-01.json
│   └── ...
├── 05-locations.json       # Phase 6 위치 카드
├── 06-meta-facts.json      # Phase 7 메타 챕터 사실
└── manifest.json           # 자동 생성, 위 파일들 통합
```

빌드 스크립트 (`build-scenario.js`)가 위 파일들을 읽어 Firestore에 업로드.

## 4. 첫 출시 시나리오 라인업 (5개)

큐레이터가 첫 6개월에 작성할 시나리오 5개.

### 4.1 *Mr. Sunshine 정서* (1894~1907 한성)

이미 위 예시들로 다룸. 무게 4/5.

### 4.2 *Hidden Court* (17세기 조선 궁중)

비교 K-드라마: 해를 품은 달, 옷소매 붉은 끝동
- 시대: 숙종 ~ 영조 시대
- 캐스팅 풀: 궁녀, 무수리, 양반가 며느리, 의녀, 향반가 평범한 사람
- 무게: 3/5
- 결말 풀: 약 35개

### 4.3 *Liberation* (1910~1945 일제강점기)

비교 K-드라마: 미스터 션샤인 후반, 동주, 암살
- 시대: 1910~1945
- 캐스팅 풀: 만주 독립군, 한성 비밀 활동가, 친일파 양심, 일본 유학 시인, 평범한 농민
- 무게: 5/5 (가장 무거움)
- 결말 풀: 약 40개

### 4.4 *Reply 1988 정서* (1988 쌍문동)

비교 K-드라마: 응답하라 1988, 응답하라 1994
- 시대: 1986~1994 서울 골목
- 캐스팅 풀: 평범한 학생, 깊은 친구, 천재 (바둑/음악), 코믹한 친구, 어른의 시점
- 무게: 2/5
- 결말 풀: 약 30개

### 4.5 *Idol Trainee* (2010s K-pop)

비교 K-드라마: 별그대 일부, K-pop 다큐
- 시대: 2010~2020 서울
- 캐스팅 풀: 데뷔 아이돌, 데뷔 못한 연습생, 매니저, 작곡가, 팬
- 무게: 2/5
- 결말 풀: 약 30개

## 6. 작성 비용 추산

| 단계 | 시나리오당 시간 | 시나리오 30개 |
|---|---|---|
| Phase 1: 컨셉 | 30분 | 15시간 |
| Phase 2: 챕터 구조 | 60분 | 30시간 |
| Phase 3: 캐스팅 조건 | 30분 | 15시간 |
| Phase 4: 명장면 setup | 90분 | 45시간 |
| Phase 5: 결말 풀 | 90분 | 45시간 |
| Phase 6: 위치 카드 | 30분 | 15시간 |
| Phase 7: 메타 챕터 | 30분 | 15시간 |
| **합계** | **6시간** | **180시간** |

180시간 = 약 한 달의 풀타임 작업. 또는 6개월간 매주 8시간씩.

**작가 외주 시:**
- 시나리오당 ~₩50만 (제휴 작가)
- 시나리오 30개 = ₩1,500만
- 1차 출시 5개는 내부 큐레이터가 직접 작성하는 것이 톤/방향 정착에 유리

## 7. 시나리오 검증 체크리스트

각 시나리오 출시 전 확인:

### 7.1 IP 안전성
- [ ] 드라마 캐릭터 이름 직접 차용 X
- [ ] 드라마 제목 직접 인용 X
- [ ] 명장면 narrative가 드라마 본 사람만 알 수 있을 정도로 특정적 X
- [ ] 변호사 검토 통과

### 7.2 역사 정확성
- [ ] 시대 사건의 연대/사실 정확
- [ ] 의상, 음식, 호칭 시대 적합
- [ ] 일본/중국/외국 묘사도 사실 기반
- [ ] 메타 챕터 출처 학술 검증

### 7.3 게임 플레이
- [ ] 모든 캐스팅이 도달 가능 (테스트 플레이로 검증)
- [ ] 모든 결말이 도달 가능
- [ ] 즉사 조건이 너무 가혹하지 않음
- [ ] 자유 입력 평가가 합리적

### 7.4 문화적 안전성
- [ ] 인종/성별/계급 차별 묘사 비판적 처리
- [ ] 일본 시장 출시 시 정치적 충돌 검토 (Liberation, Pachinko 등)
- [ ] 종교 묘사 균형
- [ ] 트리거 워닝 자동 표시

### 7.5 다국어
- [ ] 한국어 + 영어 동시 작성
- [ ] 영어 번역의 시대극 톤 적합

## 8. 시나리오 라이프사이클

```
[Draft] 작성 중
   ↓
[Review] 작가 본인 + 동료 검토
   ↓
[Legal Check] 변호사 검토 (IP 안전성)
   ↓
[Beta] 베타 사용자 100명 테스트
   ↓
[Published] 정식 출시
   ↓
(통계 기반으로)
[Refined] 명장면/결말 추가, 조건 조정
   ↓
[Archived] 출시 1년 후 트래픽 적으면 archived
```

각 단계 데이터:
- Firestore `/scenarios/{id}.status` 필드
- 빌드 스크립트가 status에 따라 게임에 노출/비노출
