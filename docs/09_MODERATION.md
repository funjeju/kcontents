# 09. 콘텐츠 모더레이션 & 안전 정책

K-Drama Life는 AI Dungeon Filtergate(2021) 같은 실패 사례에서 배워, 처음부터 안전을 설계에 박는다.

---

## 1. 모더레이션 철학

### 1.1 두 가지 위험 영역

| 영역 | 위험 | 우리 대응 |
|---|---|---|
| **큐레이션 콘텐츠** | 시나리오 자체가 IP 침해, 차별, 역사 왜곡 | 시나리오 검증 체크리스트, 변호사 검토 |
| **사용자 자유 입력** | 사용자가 부적절 콘텐츠 입력 | 4단 모더레이션 파이프라인 |

### 1.2 핵심 원칙

1. **사전 큐레이션 우선** — 자유도가 클수록 위험. 우리는 객관식 + 제한된 자유 입력만.
2. **AI Dungeon 사례 학습** — 자유 입력의 무한 자유는 모더레이션 불가능. 우리는 *시대/캐릭터 컨텍스트 안에서만* 자유.
3. **평가 게이트** — 자유 입력은 *답*이 아니라 *시도*. AI가 평가하고 시대 적합성 판단.
4. **연령별 차등** — 13~17세는 자유 입력 비활성화 (객관식만).
5. **사람 검토 백업** — AI 모더레이션이 모호하면 사람 큐로.

## 2. 큐레이션 시나리오 안전

### 2.1 시나리오 출시 전 체크

[07_SCENARIO_AUTHORING.md 7장](07_SCENARIO_AUTHORING.md) 참조. 모든 시나리오는 다음 통과 후 출시:

- [ ] IP 안전성 (변호사 검토)
- [ ] 역사 정확성 (학술 자료 출처)
- [ ] 게임 플레이 검증
- [ ] 문화적 안전성
- [ ] 다국어 톤 적합

### 2.2 무거운 콘텐츠 처리

heaviness 4~5 시나리오 (일제강점기, 5·18 등):

- 시작 전 트리거 워닝 표시
- 13~17세 사용자에게 노출 X (또는 부모 동의 필수)
- 메타 챕터 의무 (학습 효과 + 트라우마 보호)
- 폭력/고문 묘사는 *추상적*으로 (구체적 묘사 X)

### 2.3 일제강점기 콘텐츠 — 일본 시장 충돌

특별히 민감한 영역. 우리 접근:

```
원칙:
1. 사실에 기반 (학술 출처)
2. 일본인 vs 일본 정부/군부 구분
3. 회색 인물 등장 (좋은 일본인, 나쁜 한국인 모두 가능)
4. 메타 챕터에서 *"이 사건은 일본 학계에서도 점차 인정되고 있다"* 균형 톤
5. 일본 시장 진출 시:
   - 일본 시대 시나리오 우선 (에도, 메이지)
   - 식민지 시나리오는 영문/한국어 우선, 일본어는 신중한 번역
   - 일본 사용자 차단 X (균형 메시지로 접근)
```

### 2.4 IP 회피 안전선 — 5장 매트릭스

```
영역 1: 캐릭터 이름
❌ 직접 인용: "유진 초이", "고애신"
✅ 닮은꼴 X: "최진우", "김애린" (시대 적합)

영역 2: 작품 제목
❌ 마케팅에 "Mr. Sunshine"
✅ "1900s Hanseong era"

영역 3: 명대사
❌ 드라마 명대사 직접 인용
✅ 비슷한 정서의 새 대사 (AI 생성)

영역 4: 명장면
❌ 드라마 회차의 장면 그대로 재현
✅ 시대의 일반적 상황 + 비슷한 갈등 구조

영역 5: 시각 자산
❌ 드라마 스틸 사용
❌ 배우 닮은꼴 일러스트
✅ 시대 분위기 자체 일러스트
```

매 시나리오 출시 전 변호사 검토. 비용: ₩200~500만/시나리오 (5개 출시 기준 ₩1,000~2,500만).

## 3. 사용자 자유 입력 모더레이션

### 3.1 4단 파이프라인

```
사용자 자유 입력
       ↓
[1단] 키워드 차단 (즉시)
       ↓
[2단] LLM 분류 (3초)
       ↓
[3단] 컨텍스트 평가 (게임 진행과 동시)
       ↓
[4단] 사후 검증 (사람, 비동기)
```

### 3.2 1단: 키워드 차단

즉시 차단 키워드:

```
하드 차단 (입력 즉시 거부):
- 미성년자 + 성적 표현
- 자살 / 자해 구체적 묘사
- 폭력 미화 (구체적 가해 방법)
- 실명 + 사적 영역 (실존 인물 사적 헤이트)
- 극단주의 슬로건 (테러 옹호 등)

리스트 관리:
- /system/blocked-keywords 컬렉션
- 한국어 + 영어 + 일본어 + 중국어 (주요 언어)
- 변형 형태 (자모 분리, 숫자 치환) 매칭
```

차단 시 사용자에게:

```
"입력하신 내용이 우리 커뮤니티 가이드라인에 맞지 않아요.
다른 답을 작성해주시거나 객관식을 선택해주세요."

[다시 작성]  [객관식 보기]
```

부드러운 톤 — 사용자가 모욕감 안 느끼게.

### 3.3 2단: LLM 분류

[06_AI_PROMPTS.md AI-11](06_AI_PROMPTS.md) 참조. Gemini Flash로 빠르게 분류:

- ALLOW: 통과
- REVIEW: 사람 검토 큐로 (게임은 일단 진행)
- BLOCK: 차단

REVIEW 결정도 게임 자체는 진행시킨다. 이유:
- 사용자가 *"검토 중입니다"* 메시지에 짜증 X
- 만약 사후 검토에서 BLOCK 판정 시 → 그 응답을 무효화 + 사용자 알림 (보통 발생 X)

### 3.4 3단: 컨텍스트 평가

LLM이 ALLOW 판정해도, 게임 컨텍스트와 맞는지 평가 ([06_AI_PROMPTS.md AI-4](06_AI_PROMPTS.md)):

- 시대 적합성: 1900년 캐릭터가 "스마트폰" 언급 X
- 캐릭터 일관성: 도덕성 18 캐릭터가 갑자기 살인 X
- 톤 적합성: 시대극 톤에서 욕설 남발 X

부적합 시: 입력 자체는 받되, narrative 결과에 반영 X. 사용자에겐 *"당신의 답은 시대 정서와 맞지 않아 캐릭터가 다르게 행동했어요"*

### 3.5 4단: 사후 검증

비동기 사람 검토 큐:

- REVIEW 판정 입력
- 신고된 입력
- 정기 샘플링 (매일 100건)

검토자(파트타임 1명, ~₩150만/월): 
- 패턴 발견 시 키워드 리스트 업데이트
- LLM 프롬프트 보강
- 악용 사용자 계정 조치

## 4. 14세 미만 보호 (한국)

### 4.1 한국 법적 요구

한국 개인정보보호법: 14세 미만 가입 시 부모 동의 필수.

### 4.2 우리 정책

가입 시 13~17세 + 한국 거주자 = 추가 단계:

```
1. 부모 이메일 또는 전화번호 입력
2. 부모에게 동의 링크 발송
3. 부모가 동의 클릭 (해당 링크에서)
4. 동의 시점 기록 + 가입 완료
```

가입 후 미성년자 계정 추가 제약:

- heaviness 4~5 시나리오 노출 X
- 자유 입력 비활성화 (객관식만)
- 결제 차단
- 위치 카드 GPS 권한 요청 시 부모 추가 동의

### 4.3 13세 미만

- COPPA 대응: 가입 차단
- "13세 미만은 K-Drama Life를 이용할 수 없어요. 13세가 되면 다시 만나요!" 메시지

## 5. 콘텐츠 트리거 워닝

### 5.1 시나리오별 워닝

각 시나리오 시작 전:

```
[Scenario 시작 전 안내]

이 시나리오는 다음 콘텐츠를 다룹니다:
• 일제강점기 식민 통치
• 폭력 (추상적 묘사)
• 가족 이별
• 죽음 (역사적 사건)

이 게임에서 다루는 사건은 실제 역사적 사실에 기반합니다.
출처는 메타 챕터에서 확인하실 수 있습니다.

준비되셨다면 시작해주세요.

[건너뛰기]  [시작하기]
```

### 5.2 챕터별 워닝

특히 무거운 챕터 진입 시 추가 안내.

### 5.3 정신 건강 자원

게임 중 사용자 정서가 무거워질 수 있음. 설정 화면에 항상 노출:

```
[설정 → 도움이 필요하신가요?]

이 게임은 종종 무거운 주제를 다룹니다.
정서적으로 어려움을 느끼신다면:

[한국]
- 정신건강위기상담전화 1577-0199
- 청소년상담전화 1388

[Global]
- International Association for Suicide Prevention
  https://www.iasp.info/resources/Crisis_Centres/
```

## 6. 사용자 신고 시스템

### 6.1 신고 가능 대상

- 다른 사용자의 결말 카드 (공개된 것)
- 게임 내 콘텐츠 (시나리오, 명장면)
- 시스템 버그/오류

### 6.2 신고 흐름

```
[결말 카드 공유 페이지]

[이 카드 신고하기 ⚠]
   ↓
[신고 폼]
- 사유 (라디오):
  • 부적절한 표현
  • 차별/혐오
  • 시대 왜곡
  • 기타

- 추가 설명 (옵션, 200자)

[제출]
   ↓
모더레이션 큐로
```

### 6.3 처리 SLA

- 일반 신고: 48시간 내 검토
- 심각한 신고 (차별/혐오/미성년): 12시간 내 검토
- 처리 결과 신고자에게 통보

## 7. 어뷰즈 / 악용 방지

### 7.1 봇 / 자동화 방지

- 회원가입 시 reCAPTCHA v3
- 단시간 다량 시도: rate limiting
- IP 기반 + 디바이스 fingerprint

### 7.2 어뷰즈 패턴

```
의심 패턴 자동 탐지:
- 한 IP에서 1시간 50회 가입
- 비슷한 패턴 자유 입력 반복 (스팸/홍보)
- GPS 스푸핑 (5분 안에 서울→부산)
- 결말 카드 무한 공유 (홍보 도구 악용)
```

### 7.3 제재 단계

```
1차: 경고 메시지
2차: 24시간 자유 입력 차단
3차: 7일 계정 정지
4차: 영구 정지 + IP/디바이스 차단
```

## 8. 데이터 보호

### 8.1 GDPR / PIPL 대응

- 가입 시 명확한 동의 (체크박스)
- 처리 데이터 명시
- 동의 철회 시 데이터 삭제 권리
- 데이터 이동 요청 권리

### 8.2 데이터 삭제 흐름

```
사용자가 [계정 삭제] 클릭
   ↓
[확인 모달]
"정말 삭제하시겠어요? 이 작업은 되돌릴 수 없어요.
당신의 모든 인생, 카드, 컬렉션이 영구 삭제됩니다."

[취소]  [삭제 확인]
   ↓
30일 유예 기간 (그동안 복구 가능)
   ↓
30일 후 영구 삭제
   - /users/{uid} 전체 cascade 삭제
   - Cloud Storage 사용자 자산 삭제
   - 백업에서도 다음 주기에 제거
```

### 8.3 익명화

분석용 데이터는 가입 90일 후 익명화:
- email, displayName, photoURL 제거
- 통계 데이터만 보존 (인생 종류, 결말 통계)

## 10. 비상 대응

### 10.1 PR 위기 상황

만약 *"K-Drama Life가 일제 미화한다"* 같은 비판 발생 시:

```
1단계 (1시간 내): 
- 해당 시나리오/콘텐츠 일시 비활성화
- 트위터/인스타에 공식 응답 ("검토 중입니다")

2단계 (24시간 내):
- 내부 검토 (시나리오 작가 + 변호사 + 외부 전문가)
- 공식 입장문 발표

3단계 (1주 내):
- 필요 시 콘텐츠 수정/삭제
- 사용자 알림
- 재발 방지 정책 보완
```

### 10.2 AI Dungeon 식 모더레이션 위기

만약 사용자 자유 입력 + AI가 부적절 콘텐츠 생성한 사례 노출 시:

```
1단계: 즉시 자유 입력 임시 비활성화
2단계: 모든 사용자에게 공지
3단계: 모더레이션 강화 후 재오픈
4단계: 사용자에게 보상 (카드팩)
```

미리 준비된 *"비상 모드"* 토글 (`/system/config/emergency_mode`):
- 자유 입력 OFF
- 무거운 시나리오 OFF
- 결제 / 광고 일시 중단
